# PureTech——Java后端学习项目

## 项目目的

PureTech是一个Java后端学习项目，旨在记录自己学习后端技术，如果有可能帮助到其他人，也不妨是一件有意义的项目。通过构建一个完整的后端应用来学习和掌握Java后端开发的核心概念和技能。

2026年研究生考试结果已经出了，我拟录取了一所211大学。在家里休息了1个月之后，决定学习Java后端开发相关的技术，争取在暑期找一份相关实习，积累一些实战经验。

为此，我通过学习**二哥**的开源项目——技术派，来进行学习！我这里东施效颦，为项目取名为**PureTech**，纯技术的意思，哈哈哈。

### 模块拆分思考

为了更好地开始这个项目，我们需要先知道这个项目要解决什么问题，提供什么价值才行！

该项目是一个论坛项目，可以集成许多功能！为了便于理解，需要将项目进行拆分，先分出核心模块，然后再在每个模块中实现、完善功能。

论坛最核心的功能应该是让用户可以进行**学习交流**，此项目通过阅读文章和评论区实现此功能。所以，引申出**发帖**、**评论**两个核心功能。 
在此之上还可以对于文章进行归档——称为专栏，还可以由此引出点赞、收藏以及统计数据等功能。

在使用者的角度来看，可以将使用者分为不同的角色，分别是用户、作者、管理员，不同的角色，所注重的功能也不一样！用户注重于**学习交流**，作者注重于**发帖**，管理员注重于**管理**。

执行了一些操作之后（比如点赞、公告等），可能需要进行消息通知，所以还需要一个消息模块来实现这个功能。

综上，可以抽离出5个大模块：

- 用户模块
- 文章模块
- 评论模块
- 消息模块
- 通用模块

#### 用户模块

作为用户，首先就得考虑注册登录，常见的是通过账号密码或者手机号验证码进行登录，在这里我们采用个人微信登录，因为不是企业微信，所以无法进行扫码登录，
需要关注公粽号，然后将验证码发送给公粽号进行验证才行，流程稍显冗余。

然后就是用户权限，需要识别用户的身份并鉴权！核心是维护客户端使用的ID和服务器维护的用户ID之间的映射关系，服务器通过客户端请求携带的ID来获取对应用户信息，里面的包括权限等信息。

根据上述将用户分成的三个角色，我们为用户设计了**ADMIN**、**LOGIN**、**ALL**三种权限，分别对应管理员、登录用户和所有用户。每个接口都需要通过这些权限进行限制！

除了上面的基础功能外，用户还有额外的业务功能，比如关注、粉丝、个人信息维护、用户数据统计等，这些功能也需要在用户模块中进行实现。

#### 文章模块

文章模块的核心是发布和阅读，文章的内容一般不会直接展示，而是展示文章的摘要，头图等信息，所以这里的表设计需要注意一下，设计成两张表，一张是文章表，存储文章的基本信息，另一张是文章内容表，存储文章的具体内容，这样可以提高查询效率。

专栏不外乎是将文章进行分了，通过维护一张专栏文章关联表来实现，该表中存储专栏和文章的关系。

最后是用户对文章的点赞、收藏等功能，通过上面的文章表和用户表进行关联来实现的。

#### 评论模块

评论可以对文章，也可以对另一个评论，如何设计这个功能呢？此项目通过维护一个评论表来实现，每条评论都有一个父评论ID，如果父评论ID为0，说明该评论是对文章的评论，否则就是对另一个评论的回复。
要查询一条评论下的评论时，只需要查询父评论ID为该评论ID的评论即可，这样就可以实现多层级的评论了。

#### 消息模块

在系统中的一些处理完成后需要更新或者执行一些动作，但这些动作立刻执行的意义不大，可以采用消息的方式将处理完成的标志发布，通过消息队列进习惯一步处理。常见的动作有：点赞、评论、关注、通告等。

#### 通用模块

此模块更多的是提供一些抽离业务的功能，比如数据统计、数据分析、上传图片、搜索等功能，这些功能可能会被其他模块调用，所以需要将其抽离出来，放在一个通用模块中。

### maven模块拆分

项目采用 Maven 多模块结构进行组织，将不同职责的代码拆分到独立模块中，通过根目录的父 pom 统一管理依赖版本和模块构建顺序。本质上仍是单体应用，最终由 puretech-web 模块聚合所有子模块打包为一个可运行的 jar 包部署。

| 模块                 | 职责                        | 依赖关系                              |
|--------------------|---------------------------|-----------------------------------|
| `puretech-api`     | 公共数据结构，实体类、DTO、VO、枚举      | 不依赖任何内部模块                         |
| `puretech-core`    | 通用工具类、基础组件、全局异常、常量        | 依赖 `puretech-api`                 |
| `puretech-service` | 业务逻辑、数据库操作、Mapper         | 依赖 `puretech-api`、`puretech-core` |
| `puretech-web`     | 启动类、Controller、拦截器、MVC 配置 | 依赖所有模块                            |

请求流动示意图：

``` md
前端请求
   ↓
puretech-web     # 接收 HTTP 请求
   ↓
puretech-service # 处理业务逻辑、操作数据库
   ↓
puretech-api     # 公共数据结构流转
   ↑
puretech-core    # 工具类支撑
```
